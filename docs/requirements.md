# Help A Stray Requirements

## Functional Requirements

FR1: Users shall be able to view animals available for adoption.

FR2: Users shall be able to search animals by name.

FR3: Users shall be able to filter animals by species and adoption status.

FR4: Users shall be able to view detailed animal information.

FR5: Users shall be able to submit adoption applications.

FR6: Administrators shall be able to log into the administration area.

FR7: Administrators shall be able to view adoption applications.

FR8: Administrators shall be able to approve or reject applications.

FR9: Administrators shall be able to add animal profiles.

FR10: Administrators shall be able to edit animal profiles.

FR11: Administrators shall be able to delete animal profiles.

FR12: Administrators shall be able to upload animal images.

## Non-Functional Requirements

NFR1: The application shall be accessible through a web browser.

NFR2: The application shall provide a responsive user interface.

NFR3: The application shall use a relational database.

NFR4: The application shall validate user input.

NFR5: The application shall provide secure administrator authentication.

## Security Requirements

SR1: Passwords shall be stored using password hashing.

SR2: SQL queries shall use prepared statements.

SR3: Administrator pages shall require authenticated sessions.

SR4: Uploaded files shall be restricted to supported image formats.
