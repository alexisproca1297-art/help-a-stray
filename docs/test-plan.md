# Help A Stray Test Plan

## Purpose

The purpose of testing is to verify that the Help A Stray application functions correctly and meets the project requirements. Testing was carried out throughout the development process to ensure that all public and administrator features operated as expected.

## User Authentication Testing

The administrator login functionality was tested using both valid and invalid credentials. Successful login attempts redirected the administrator to the dashboard, while invalid credentials displayed an error message. Additional testing confirmed that administrator pages could not be accessed without authentication.

## Animal Management Testing

The animal management module was tested by adding, editing and deleting animal profiles. Tests confirmed that animal records were correctly stored in the database and displayed on the public website. Image upload functionality was also tested to ensure that uploaded images were stored and displayed correctly.

## Animal Search and Filtering Testing

The search functionality was tested using different animal names to verify that matching results were returned. The filtering functionality was tested using species and adoption status filters to ensure that only relevant animals were displayed.

## Adoption Application Testing

The adoption application form was tested using valid user data. Successful submissions were stored in the database and became visible within the administrator area. Validation checks ensured that required fields could not be left empty.

## Application Review Testing

The administrator application management system was tested by reviewing submitted applications and changing their status to Approved, Rejected and Pending. Testing confirmed that status changes were stored correctly and displayed throughout the system.

## Image Upload Testing

Image upload functionality was tested using supported file formats including JPG, PNG, GIF and WEBP. Uploaded images were successfully displayed on the homepage, animal listing page and animal details page.

## Results

All major features were tested successfully. No critical errors were identified during testing. Minor user interface improvements were implemented throughout development to improve usability and presentation.

## Future Testing

Future work could include usability testing with real users, cross-browser compatibility testing and performance testing under larger datasets.
